## 💡 KEY DECISIONS KIRO MADE (Understand Why)

### 1. Analysis Page Independence
**Decision**: Analysis works WITHOUT user profile/onboarding  
**Why**: Reduce friction, allow anonymous usage  
**Implementation**: Default to 10 R22 CSE subjects (hardcoded fallback)  
**File**: `frontend/src/pages/Analysis.tsx`  
**Doc**: `ONBOARDING_REMOVED_FROM_ANALYSIS.md` (if exists)

### 2. Theory Subjects Only
**Decision**: Only 10 theory subjects (excluded labs, projects)  
**Why**: Focus on exam papers (core MVP use case)  
**List**: A6CS05, A6IT02, A6CS28, A6CS07, A6BS03, A6HS08, A6CS08, A6CS09, A6CS11, A6CS13  
**Doc**: `R22_INGESTION_COMPLETE.md`

### 3. Classification Algorithm
**Decision**: Keyword-based Jaccard similarity (conservative)  
**Why**: Accuracy over coverage, prevent false positives  
**Coverage**: 47.7% (3,433/7,193 questions)  
**Future**: LLM fallback for edge cases  
**File**: `backend/app/analysis/topic_classifier.py`

### 4. Authentic DOCX Downloads
**Decision**: Store original MLRIT DOCX files, never serve AI-generated as "original"  
**Why**: Academic integrity, authenticity  
**Storage**: Supabase Storage bucket `paper` (77 files, ~40MB)  
**Path**: `R22/CSE/filename.docx`  
**File**: `backend/app/api/download.py`

### 5. Multi-Worker for Scale
**Decision**: Gunicorn with 9 workers (not single Uvicorn process)  
**Why**: Handle 300-400 concurrent users (10x capacity boost)  
**Current**: Without this, only 20-30 users supported  
**File**: `backend/gunicorn.conf.py`  
**Doc**: `PRODUCTION_READINESS_AUDIT.md`

### 6. Error Boundaries & Loading States
**Decision**: Comprehensive error handling and loading feedback  
**Why**: Professional UX, prevent white screen crashes  
**Implementation**: ErrorBoundary wraps all routes, loading states on all async ops  
**Files**: `frontend/src/components/ErrorBoundary.tsx`, `LoadingState.tsx`

### 7. React Query Caching (Planned)
**Decision**: Add React Query for global state caching  
**Why**: 70% reduction in network requests  
**Status**: Library installed, needs implementation  
**Task**: For you to implement during deployment

### 8. Exam Year as NULL
**Decision**: Leave 77/80 papers with `exam_year=NULL`  
**Why**: Paper filenames don't contain year info, accurate to show "Past Paper"  
**Not a bug**: Correct handling of missing data  
**Future**: Manual entry if year info becomes available
